# /kickoff — 作業開始の儀式

のんとの作業を始める前の準備運動やで。以下を順番に実行すること。

## 手順

1. **日時確認**：`TZ=Asia/Tokyo date` で日本時間の今日の日付を確認する。以降の日付記載はすべて JST。

2. **正本を読む**（あるものだけでよい。なければ「まだない」と報告）：
   - `docs/current_state.md` — 北極星ファイル。最新の正本
   - `docs/ai_handoff.md` — 開発ルール・触ってはいけない領域・事故パターン
   - `docs/handoff/` 配下の最新の引き継ぎメモ（前回の `/wrapup` が作ったもの）
   - リポジトリ直下の CLAUDE.md（あれば共通設定より優先）

3. **リポの現状確認**：
   - `git log --oneline -10` と `git status` で、前回からの変更と作業中のものがないか確認
   - 引き継ぎメモに書かれた「次回やること」と、実際のコードの状態に食い違いがないか確認する。食い違いがあれば正直にのんに伝える（コードの現物が優先）

4. **のんへの報告**：以下を明るく分かりやすくまとめて伝える
   - 📍 **いまどこ**：プロジェクトの現在地（current_state.md の要約）
   - 📝 **前回の続き**：引き継ぎメモにあった「次回やること」
   - ⚠️ **気になること**：食い違い・未解決の課題があれば
   - 🎯 **今日の提案**：今日やるとよさそうなことの提案（のんが別の希望を言ったらそちらを優先）

## 守ること

- 正本ドキュメントが1つもないリポジトリだった場合は、「北極星ファイルを作るところから始めよか」と提案する
- 読んだ内容を長々と貼り付けない。のんが30秒で読める要約にする
- ここで勝手に作業を始めない。計画をのんと合意してから動く
